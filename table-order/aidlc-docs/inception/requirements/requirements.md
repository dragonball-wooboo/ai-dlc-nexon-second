# 테이블오더 서비스 요구사항 정의서

## Intent Analysis

| 항목 | 내용 |
|------|------|
| **사용자 요청** | 소규모 매장용 테이블오더 서비스 구축 |
| **요청 유형** | New Project (신규 프로젝트) |
| **범위** | System-wide (전체 시스템 구축) |
| **복잡도** | Moderate (다수 기능이지만 명확한 요구사항) |
| **팀 구성** | 비개발자 4인 |

## 기술 스택 결정

| 영역 | 선택 | 비고 |
|------|------|------|
| 백엔드 | Node.js + Express (TypeScript) | REST API 서버 |
| 프론트엔드 (고객용) | React (TypeScript) | 별도 앱 |
| 프론트엔드 (관리자용) | React (TypeScript) | 별도 앱 |
| 데이터베이스 | SQLite | 파일 기반, 경량 |
| 배포 환경 | 로컬 개발 환경 | 배포 없음 |
| 이미지 관리 | 로컬 파일 시스템 저장 + 서빙 | 업로드 기능 포함 |
| 실시간 통신 | Server-Sent Events (SSE) | 주문 모니터링용 |

## 아키텍처 구성

```
+------------------+     +------------------+     +------------------+
|  고객용 앱       |     |  관리자용 앱      |     |                  |
|  (React SPA)    |     |  (React SPA)     |     |   SQLite DB      |
|  Port: 3001     |     |  Port: 3002      |     |   (파일 기반)     |
+--------+---------+     +--------+---------+     +--------+---------+
         |                         |                        ^
         |        REST API         |                        |
         +------------+------------+                        |
                      |                                     |
                      v                                     |
              +-------+--------+                            |
              |  백엔드 서버    +----------------------------+
              |  (Express)     |
              |  Port: 3000    |
              |  + SSE         |
              |  + Static Files|
              +----------------+
```

- **고객용 앱**: 테이블 태블릿에서 동작하는 주문 인터페이스
- **관리자용 앱**: 매장 운영자가 사용하는 관리 대시보드
- **백엔드 서버**: API 처리, SSE 실시간 통신, 이미지 파일 서빙

## 기능 요구사항 (Functional Requirements)

### FR-1: 고객용 기능

#### FR-1.1: 테이블 태블릿 자동 로그인
- 매장 식별자 + 테이블 번호 + 비밀번호로 초기 설정 (1회)
- 로그인 정보 로컬 저장 (localStorage)
- 이후 자동 로그인

#### FR-1.2: 메뉴 조회
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세: 메뉴명, 가격, 설명, 이미지
- 카드 형태 레이아웃
- 터치 친화적 UI (최소 44x44px 버튼)

#### FR-1.3: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- localStorage 저장 (새로고침 유지)
- 주문 확정 시에만 서버 전송

#### FR-1.4: 주문 생성
- 주문 내역 최종 확인 후 확정
- 성공 시: 주문 번호 표시 → 5초 후 메뉴 화면 리다이렉트 → 장바구니 비우기
- 실패 시: 에러 메시지 + 장바구니 유지
- 주문 정보: 매장ID, 테이블ID, 메뉴 목록, 총 금액, 세션ID

#### FR-1.5: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)

### FR-2: 관리자용 기능

#### FR-2.1: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반 인증
- 16시간 세션 유지
- bcrypt 비밀번호 해싱

#### FR-2.2: 실시간 주문 모니터링
- SSE 기반 실시간 주문 업데이트 (2초 이내)
- 테이블별 그리드/카드 레이아웃
- 각 카드: 테이블 번호, 총 주문액, 최신 주문 미리보기
- 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조

#### FR-2.3: 테이블 관리
- 테이블 초기 설정 (번호, 비밀번호)
- 주문 삭제 (확인 팝업 → 즉시 삭제 → 총액 재계산)
- 테이블 이용 완료 (세션 종료 → 주문 이력 이동 → 리셋)
- 과거 주문 내역 조회 (날짜 필터링)

#### FR-2.4: 메뉴 관리
- CRUD: 메뉴 등록/조회/수정/삭제
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지
- 이미지 파일 업로드 (로컬 저장)
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증

## 비기능 요구사항 (Non-Functional Requirements)

### NFR-1: 성능
- SSE 주문 알림 2초 이내 전달
- 소규모 환경 (1매장, 10테이블 이하) 기준 최적화

### NFR-2: 사용성
- 터치 친화적 UI (44x44px 최소 터치 영역)
- 직관적 네비게이션
- 명확한 시각적 피드백 (성공/실패)

### NFR-3: 데이터 무결성
- 주문 데이터 영속성 보장 (SQLite)
- 장바구니 클라이언트 측 영속성 (localStorage)
- 세션 기반 주문 격리

### NFR-4: 유지보수성
- TypeScript 사용으로 타입 안전성 확보
- 명확한 프로젝트 구조
- 코드 주석 및 문서화

---

## 제외 범위 (Out of Scope)

요구사항 constraints.md에 따라 다음은 구현하지 않음:
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS, 2FA)
- 이미지 리사이징/최적화
- 알림 시스템 (푸시, SMS, 이메일, 소리)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석, 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달, POS, 소셜, 지도, 번역)
