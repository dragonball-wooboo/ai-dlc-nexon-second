# 사용자 스토리

## 고객 스토리 (Customer Stories)

### US-C01: 테이블 자동 로그인
**As a** 고객  
**I want to** 태블릿을 켜면 자동으로 로그인되어 있기를  
**So that** 별도 인증 없이 바로 주문할 수 있다

**Acceptance Criteria:**
- 초기 설정된 테이블은 자동 로그인 상태 유지
- 새로고침해도 로그인 유지 (localStorage)
- 로그인 실패 시 초기 설정 화면 표시

---

### US-C02: 메뉴 조회
**As a** 고객  
**I want to** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인하기를  
**So that** 원하는 음식을 쉽게 찾아 선택할 수 있다

**Acceptance Criteria:**
- 카테고리별 메뉴 목록 표시
- 메뉴 카드에 이름, 가격, 이미지 표시
- 카테고리 간 빠른 이동 가능

---

### US-C03: 장바구니 관리
**As a** 고객  
**I want to** 메뉴를 장바구니에 담고 수량을 조절하기를  
**So that** 주문 전에 원하는 메뉴를 정리할 수 있다

**Acceptance Criteria:**
- 메뉴 추가/삭제/수량 변경 가능
- 총 금액 실시간 계산
- 새로고침해도 장바구니 유지 (localStorage)

---

### US-C04: 주문 생성
**As a** 고객  
**I want to** 장바구니의 메뉴를 주문으로 확정하기를  
**So that** 매장에서 음식을 준비받을 수 있다

**Acceptance Criteria:**
- 주문 확정 전 최종 내역 확인
- 성공 시 주문 번호 표시 후 5초 뒤 메뉴 화면 이동
- 성공 시 장바구니 자동 비우기
- 실패 시 에러 메시지 표시, 장바구니 유지

---

### US-C05: 주문 내역 조회
**As a** 고객  
**I want to** 현재 세션의 주문 내역과 상태를 확인하기를  
**So that** 주문이 잘 접수되었는지, 준비 상태를 알 수 있다

**Acceptance Criteria:**
- 현재 테이블 세션 주문만 표시
- 주문별 번호, 시각, 메뉴, 금액, 상태 표시
- 시간 순 정렬

---

## 관리자 스토리 (Admin Stories)

### US-A01: 매장 로그인
**As a** 관리자  
**I want to** 매장 식별자와 비밀번호로 로그인하기를  
**So that** 매장 관리 시스템에 접근할 수 있다

**Acceptance Criteria:**
- 매장ID + 사용자명 + 비밀번호 입력
- JWT 토큰 발급, 16시간 세션 유지
- 새로고침해도 세션 유지

---

### US-A02: 실시간 주문 모니터링
**As a** 관리자  
**I want to** 들어오는 주문을 실시간으로 확인하기를  
**So that** 즉시 주문을 처리할 수 있다

**Acceptance Criteria:**
- SSE로 2초 이내 신규 주문 표시
- 테이블별 그리드 카드 레이아웃
- 각 카드에 테이블 번호, 총 주문액, 최신 주문 미리보기
- 신규 주문 시각적 강조

---

### US-A03: 주문 상태 변경
**As a** 관리자  
**I want to** 주문 상태를 대기중/준비중/완료로 변경하기를  
**So that** 주문 처리 진행 상황을 관리할 수 있다

**Acceptance Criteria:**
- 주문 카드에서 상태 변경 가능
- 상태: 대기중 → 준비중 → 완료

---

### US-A04: 테이블 초기 설정
**As a** 관리자  
**I want to** 테이블 번호와 비밀번호를 설정하기를  
**So that** 고객 태블릿이 자동 로그인할 수 있다

**Acceptance Criteria:**
- 테이블 번호 + 비밀번호 설정
- 설정 완료 시 해당 테이블 활성화

---

### US-A05: 주문 삭제
**As a** 관리자  
**I want to** 잘못된 주문을 삭제하기를  
**So that** 정확한 주문 내역을 유지할 수 있다

**Acceptance Criteria:**
- 삭제 전 확인 팝업 표시
- 삭제 후 테이블 총 주문액 재계산
- 성공/실패 피드백 표시

---

### US-A06: 테이블 이용 완료
**As a** 관리자  
**I want to** 고객 퇴장 시 테이블 세션을 종료하기를  
**So that** 다음 고객이 깨끗한 상태에서 시작할 수 있다

**Acceptance Criteria:**
- 확인 팝업 후 세션 종료
- 주문 내역 과거 이력으로 이동
- 테이블 주문 목록 및 총액 0으로 리셋

---

### US-A07: 과거 주문 내역 조회
**As a** 관리자  
**I want to** 테이블별 과거 주문 내역을 조회하기를  
**So that** 이전 매출과 주문 이력을 확인할 수 있다

**Acceptance Criteria:**
- 테이블별 과거 주문 목록 (시간 역순)
- 날짜 필터링 기능
- 주문 정보: 번호, 시각, 메뉴, 금액, 이용 완료 시각

---

### US-A08: 메뉴 관리 (CRUD)
**As a** 관리자  
**I want to** 메뉴를 등록/수정/삭제하기를  
**So that** 매장 메뉴를 최신 상태로 유지할 수 있다

**Acceptance Criteria:**
- 메뉴 등록: 메뉴명, 가격, 설명, 카테고리, 이미지 업로드
- 메뉴 수정: 모든 필드 수정 가능
- 메뉴 삭제: 확인 후 삭제
- 필수 필드 검증, 가격 범위 검증
- 노출 순서 조정 가능

---

## 스토리 의존성

```
US-A04 (테이블 설정) → US-C01 (자동 로그인) → US-C02 (메뉴 조회)
US-A08 (메뉴 관리) → US-C02 (메뉴 조회) → US-C03 (장바구니) → US-C04 (주문)
US-C04 (주문) → US-A02 (주문 모니터링) → US-A03 (상태 변경)
US-C04 (주문) → US-C05 (내역 조회)
US-A03 (상태 변경) → US-A06 (이용 완료) → US-A07 (과거 내역)
US-A02 (주문 모니터링) → US-A05 (주문 삭제)
```

## INVEST 검증

| 기준 | 충족 |
|------|------|
| Independent | ✅ 각 스토리 독립 구현 가능 (의존성은 있으나 독립 테스트 가능) |
| Negotiable | ✅ 세부 구현은 협의 가능 |
| Valuable | ✅ 각 스토리가 사용자에게 직접적 가치 제공 |
| Estimable | ✅ 범위가 명확하여 추정 가능 |
| Small | ✅ 각 스토리가 단일 기능 단위 |
| Testable | ✅ Acceptance Criteria로 테스트 가능 |
